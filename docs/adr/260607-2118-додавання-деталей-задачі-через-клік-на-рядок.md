## ADR Додавання деталей задачі через клік на рядок

## Context and Problem Statement
У Tauri-застосунку `nitra/task` відображався список задач без можливості переглянути деталі окремої задачі. Файл `task.md` кожної задачі містить її опис, але UI не мав механізму для його відкриття.

## Considered Options
* Drawer (side panel) із markdown-вмістом
* Dialog (модальне вікно) із markdown-вмістом
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Dialog із markdown-вмістом", because drawer із `maximized` атрибутом займав весь екран і було вирішено переключитись на dialog без `maximized` для нормального вигляду.

### Consequences
* Good, because transcript фіксує очікувану користь: клік на рядок задачі відкриває діалог із рендереним markdown-вмістом `task.md`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## ADR Спільний файл документа через символьне посилання між репозиторіями

## Context and Problem Statement
Файл `думка.MD` із дизайном деталей задач зберігається в `cursor/docs/` і потрібен у двох проектах (`nitra/cursor` та `nitra/task`). Зберігати дублікати небажано; файл редагується часто, тому підходи з публікацією через npm непрактичні.

## Considered Options
* Відносний symlink між репозиторіями
* Абсолютний symlink
* Переміщення до `@nitra/cursor` npm-пакету з публікацією нової версії
* Локальна `file:` залежність у `package.json`
* Відносний `@`-імпорт у `CLAUDE.md`

## Decision Outcome
Chosen option: "Відносний symlink між репозиторіями", because `@`-імпорт за межі репозиторія не підтримується Claude Code (повертає `Failed to read file: ../cursor/docs/думка.MD`), а публікація npm при кожному редагуванні документа — надто трудомістка.

### Consequences
* Good, because файл існує в одному місці (`cursor/docs/думка.MD`), symlink має відносний шлях `../../cursor/docs/думка.MD` і працює на будь-якому комп'ютері де обидва репо клоновані поруч під спільною батьківською директорією.
* Bad, because symlink зламається якщо `cursor` і `task` клоновані в різні директорії (без спільного батька `nitra/`).

## More Information
- Symlink: `task/docs/думка.MD -> ../../cursor/docs/думка.MD`
- Команда створення: `ln -s ../../cursor/docs/думка.MD /Users/vitaliytv/www/nitra/task/docs/думка.MD`
- Claude Code не підтримує `@`-шляхи за межі git-репозиторію — підтверджено помилкою `Failed to read file: ../cursor/docs/думка.MD`
- Для підключення в `task/CLAUDE.md` використовується `@docs/думка.MD` (резолвиться через symlink)
