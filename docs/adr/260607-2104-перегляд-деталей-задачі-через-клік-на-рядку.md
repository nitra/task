## ADR Читання `task.md` через Tauri-команду на бекенді

## Context and Problem Statement
Frontend-код потребував доступу до вмісту файлу `task.md` з довільної директорії tasks. Необхідно вирішити, де виконувати читання файлової системи: у Rust-бекенді через Tauri IPC чи безпосередньо у Vue-коді.

## Considered Options
* Tauri-команда `read_task` на Rust-бекенді
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Tauri-команда `read_task` на Rust-бекенді", because весь існуючий доступ до файлової системи в проєкті вже реалізований через Tauri IPC (`scan_tasks`, `find_tasks_dir`), нова команда слідує тій самій конвенції.

### Consequences
* Good, because transcript фіксує очікувану користь: `cargo check` пройшов чисто (`Finished dev profile in 0.64s`), команда зареєстрована поряд з існуючими обробниками.
* Bad, because функція недоступна у звичайному браузері (тільки в Tauri webview) — це підтверджено в transcript: `Cannot read properties of undefined (reading 'invoke')` при відкритті через Chrome.

## More Information
- `app/src-tauri/src/lib.rs` — функція `fn read_task(tasks_dir: String, task_path: String) -> Result<String, String>` читає `<tasks_dir>/<task_path>/task.md` через `std::fs::read_to_string`
- Зареєстрована в `tauri::generate_handler![scan_tasks, find_tasks_dir, read_task]`
- Перевірка: `cargo check --manifest-path src-tauri/Cargo.toml`
