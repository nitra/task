# omlx як локальний офлайн LLM-провайдер для тул-сурфейсу

**Status:** Accepted
**Date:** 2026-06-15

## Context and Problem Statement

Проєкт `task` (Tauri + Vue 3) потребує LLM-агента для headless виконання тулів (scan, workspaces, create) без UI. Потрібно обрати провайдера і протокол для адаптера `app/src/tool/llm.js`, задовольнивши вимоги приватності та офлайн-доступності.

## Considered Options

- Claude (Anthropic) через `tauri-plugin-http` — хмарний, потребує ключа і мережі
- Локальна офлайн-модель через omlx (OpenAI-сумісний MLX-сервер) + Gemma 4 E4B
- Провайдер-агностичний інтерфейс з кількома бекендами — відкладено як передчасне для MVP

## Decision Outcome

Chosen option: "Локальна офлайн-модель через omlx (OpenAI-сумісний MLX-сервер) + Gemma 4 E4B", because вимоги — приватність і офлайн-доступність; omlx говорить OpenAI API, тому `tauri-plugin-http` підходить без змін, а `toolManifest()` генерує стандартний OpenAI function-calling формат із того самого `src/tool/catalog.js`. MCP залишено як ціль після MVP.

### Consequences

- Good, because end-to-end цикл підтверджено наживо: `gemma-4-e2b-it-4bit` через omlx самостійно побудувала tool_call → `dispatch` → `mt-scanner create` → файли `demo-llm/task.md`+`demo-llm/a.md` реально створено. Адаптер `createOpenAiChat` injection-сумісний: Tauri (`tauri-plugin-http` fetch), CLI (`node fetch`), тести (mock).
- Bad, because `gemma-4-e4b-it-OptiQ-4bit` впала з HTTP 507 — перевищено memory ceiling 11.84 GB (~17.89 GB потрібно). Мала 4B-модель нестабільна в tool-use: рекомендовано набір ≤3 тулів і 1 tool-call за хід.
- Neutral, because per-verb spawn для CLI-транспорту MVP: handlers спавнять verb-и `mt-scanner` (`scan`, `workspaces`, `create`) напряму — mt-репо не змінювалось; уніфікований `mt-scanner exec '<json>'` залишено наступним кроком. Спільна архітектура `catalog → dispatch → {ok,output}` є платформо-незалежним ядром; per-stack реалізація транспорту делегується в профільні правила (`n-tauri`, `n-vue`, `n-capacitor`).

## More Information

Файли: `app/src/tool/llm.js` (`createOpenAiChat`, `runAgent`), `app/src/composables/use-omlx.js`, `app/bin/task.mjs` (команда `task agent "<prompt>"`). Конфіг: `OMLX_BASE_URL` (дефолт `http://127.0.0.1:8000/v1`), `OMLX_MODEL` (дефолт `gemma-4-e4b-it-OptiQ-4bit`), `OMLX_API_KEY`. CLI-транспорт: функція `spawnVerb` у `app/bin/task.mjs`, резолвер шукає `mt/target/release/mt-scanner` або env `MT_SCANNER_BIN`. Commit: `e36a107`.
