# OMLX API-ключ: env замість localStorage у Tauri-застосунку

**Status:** Accepted
**Date:** 2026-06-15

## Context and Problem Statement

У Tauri-застосунку API-ключ для локального omlx-сервера (OpenAI-compatible MLX) зберігався в `localStorage` webview і мав вводитися вручну через ⚙-панель `AgentDialog`. Ключ вже існував у конфігурації системи (`~/.zshrc`). Потрібно позбутися дублювання і ручного введення per-app.

## Considered Options

- Зберігати ключ у `localStorage` (попередній підхід)
- Читати ключ з env-змінних через нову Tauri-команду `invoke('omlx_config')` при монтуванні компонента

## Decision Outcome

Chosen option: "Читати ключ з env-змінних через нову Tauri-команду `invoke('omlx_config')`", because env є єдиним джерелом правди без дублювання — ключ вже лежить у `~/.zshrc`, а CLI (`app/bin/task.mjs`) вже читав `OMLX_API_KEY` з env до цієї зміни.

### Consequences

- Good, because ключ більше не потрібно вводити вручну в UI; `loadEnv()` перекриває localStorage-дефолт при монтуванні `AgentDialog`.
- Bad, because якщо env-змінна відсутня (наприклад, застосунок запущено з Finder/Dock без `~/.zshrc`) — поле залишається порожнім і сервер поверне `401`.

## More Information

Змінені файли: `src-tauri/src/lib.rs` — нова команда `omlx_config` читає `OMLX_BASE_URL`, `OMLX_MODEL`, `OMLX_API_KEY` з `std::env`; `src/composables/use-omlx.js` — `apiKey` прибрано з localStorage, додано `loadEnv()` з `invoke('omlx_config')`; `src/components/AgentDialog.vue` — `onMounted(() => loadEnv())`. Налаштування: `export OMLX_API_KEY="ключ"` у `~/.zshrc`, потім перезапустити термінал і `tauri dev`.

## Update 2026-06-15

**Прогрес: `~/.omlx/settings.json` як канонічне джерело конфігурації.** Команду `omlx_config` розширено: спочатку читається `~/.omlx/settings.json` (поля `server.host`, `server.port`, `auth.api_key`), а env-змінні `OMLX_*` є override. Файл `~/.omlx/settings.json` вже існує на кожній машині де встановлено omlx — єдине джерело без потреби в launchd/plist.

Обмеження env-override: GUI `.app`, запущений із Finder/Dock, не бачить `~/.zshrc`, тому `launchctl setenv` або LaunchAgent потрібні лише для env-override; читання `~/.omlx/settings.json` напряму цього обмеження не має. Перевірено: `cargo check` і `vitest run` (36 тестів) — зелені.
