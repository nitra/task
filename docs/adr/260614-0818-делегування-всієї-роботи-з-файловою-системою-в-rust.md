## ADR Семантика `--dep`: топологічне ребро без `ref:` на момент створення

## Context and Problem Statement
При виклику `create_task` з параметром `deps` треба вирішити, що записувати у файли `deps/<id>.md`: порожній файл (тільки ребро залежності) або вміст `ref:` (вказівник на конкретний вихідний файл депа).

## Considered Options
* Порожній `deps/<id>.md` — оголошує топологічне ребро; `ref:` дописується пізніше.
* `deps/<id>.md` з `ref:` — одразу wire data-flow.

## Decision Outcome
Chosen option: "Порожній `deps/<id>.md`", because у момент створення залежний вузол ще не виконаний; конкретний факт-файл (`fact_NNN.md`), на який вказував би `ref:`, не існує — запис `ref:` зараз = dangling reference. `ref:` дописується окремою authoring-дією після того, як деп resolved.

### Consequences
* Good, because transcript фіксує очікувану користь: немає dangling refs; `create_task` залишається pure-topological, без здогадок про майбутні виходи.
* Bad, because transcript не містить підтверджених негативних наслідків.
