# Редизайн UI до простого, лаконічного, технологічного стилю

**Status:** Accepted
**Date:** 2026-06-13

## Context and Problem Statement

Існуючий UI Quasar + Vue мав важкий яскраво-синій elevated заголовок і неоднорідні індикатори стану, що суперечило вимозі «простого, лаконічного, технологічного, збалансованого» дизайну. Стан задачі відображався по-різному у `TaskGraph.vue` та `TaskNodeItem.vue` через відсутність спільного джерела конфігурації.

## Considered Options

- Редизайн з плоским заголовком, єдиним `state-config.js` та уніфікованими state-pill компонентами
- Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Редизайн з плоским заголовком і централізованим `state-config.js`", because користувач задав напрямок «простий, лаконічний, технологічний, збалансований»; вилучено elevated заголовок на користь плоского із hairline-роздільником, палітра станів централізована у `state-config.js`.

### Consequences

- Good, because `state-config.js` гарантує консистентність кольорів та іконок між `TaskGraph.vue` та `TaskNodeItem.vue`.
- Good, because dark і light теми відрендерились без артефактів у headless Chrome.
- Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- Змінені файли: `app/src/App.vue`, `app/src/components/TaskGraph.vue`, `app/src/components/TaskNodeItem.vue`
- Новий файл: `app/src/state-config.js` — іконки, акцентні кольори та мітки для кожного стану
- Старі константи `STATE_COLOR` та `STATE_CFG` вилучені; `selectedCfg` переведено на `state-config.js`
- Перевірка: `bunx vite build` — 86 модулів, без помилок; headless Chrome `--window-size=1100,820` і `--window-size=1100,500`
- Поріг jscpd: дублюваний CSS state-pill (~23 рядки) не перевищує `minLines: 25` зі `.jscpd.json`
