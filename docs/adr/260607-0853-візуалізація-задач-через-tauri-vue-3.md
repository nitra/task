## ADR Візуалізація задач через Tauri + Vue 3

## Context and Problem Statement

У проекті `nitra/task` задачі зберігаються як `task.md`-файли у директоріях `tasks/*/`. Потрібно відображати їх стан та залежності у вигляді дерева, де стан кожного вузла визначається не явним полем, а наявністю певних файлів у директорії задачі.

## Considered Options

* Сканування файлової системи на стороні Rust через Tauri-команду з деривацією стану з файлів
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Сканування файлової системи на стороні Rust через Tauri-команду з деривацією стану з файлів", because архітектурний документ (`думка.MD`) визначає, що стан вузла визначається наявністю файлів (`outputs_*.md`, `run_*.md`, `pending-audit_*.md`, `invalidated`, `.worktrees/`) — явного поля стану немає, тому логіка деривації живе в Rust-бекенді, а фронтенд відображає готові дані.

### Consequences

* Good, because transcript фіксує очікувану користь: Vue-компоненти (`TaskGraph.vue`, `TaskNodeItem.vue`) отримують готові `TaskNode`-структури з уже обчисленим `state`, не містять логіки деривації.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- Tauri-команди: `scan_tasks(tasks_dir: String)` і `find_tasks_dir()` у `app/src-tauri/src/lib.rs`
- Стан деривується у Rust: перевіряються файли `outputs_*.md` → `resolved`; `run_*.md` без `outputs_*` → `failed`; `pending-audit_*.md` без результату → `pending_audit`; sentinel `invalidated` → `invalidated`; активний worktree у `.worktrees/` → `running`; `plan_*.md` без `mode: human` → `waiting`; інакше → `human_pending`
- Композитні вузли визначаються наявністю дочірніх директорій; їх стан агрегується з дочірніх
- `find_tasks_dir()` обходить дерево вгору від `cwd`, шукаючи `.n-cursor.json` або `tasks/*/task.md`
- Іконки (`material-symbols-outlined`) вимагають префікса `sym_o_` у Quasar: `sym_o_schedule`, `sym_o_check_circle` тощо
- Для збірки Tauri потрібні файли-заглушки у `app/src-tauri/icons/` (RGBA PNG + ICO + ICNS)
- Компоненти: `app/src/components/TaskGraph.vue`, `app/src/components/TaskNodeItem.vue`
- Запуск: `bun run start` з кореня проекту
