# Перегляд деталей задачі через клік на рядок

**Status:** Accepted
**Date:** 2026-06-07

## Context and Problem Statement

Застосунок `task` відображав граф задач у вигляді списку, але не мав способу переглянути вміст окремої задачі (`task.md`) безпосередньо з UI, не відкриваючи файли в редакторі.

## Considered Options

- Drawer-панель праворуч (side drawer) з `maximized` prop та вмістом `task.md`
- Модальний `q-dialog` по кліку на рядок задачі (фіксована ширина, fade-анімація)
- Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Модальний `q-dialog` по кліку на рядок", because початково вибраний `maximized` drawer займав весь екран і був відхилений; діалог з шириною 640px (`max-width: 90vw`) та `transition-show="fade"` обраний як компактніший.

### Consequences

- Good, because функціонал запрацював у рідному Tauri-вікні після перебудови Rust-бінарника; клік на рядок відкриває відформатований вміст `task.md`.
- Bad, because `invoke` з `@tauri-apps/api/core` не працює у звичайному браузері (`Cannot read properties of undefined (reading 'invoke')`) — тестування UI можливе лише у рідному Tauri-вікні.

## More Information

- `app/src/components/TaskNodeItem.vue` — `@click.stop` на рядку задачі, emit `select` вгору до `TaskGraph.vue`
- `app/src/components/TaskGraph.vue` — `q-dialog v-model="drawerOpen"`, логіка `openTask()`, CSS `.task-detail-card`; рендеринг через `marked@18.0.5` (`bun add marked` у директорії `app/`)
- `app/src-tauri/src/lib.rs` — нова команда `fn read_task(tasks_dir: String, task_path: String) -> Result<String, String>`: читає `<tasks_dir>/<task_path>/task.md` через `std::fs::read_to_string`; зареєстрована в `tauri::generate_handler![scan_tasks, find_tasks_dir, read_task]`
- Перевірка компіляції: `cargo check --manifest-path src-tauri/Cargo.toml` → `Finished dev profile in 0.64s`
