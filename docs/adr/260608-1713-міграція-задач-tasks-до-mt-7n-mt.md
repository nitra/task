# Міграція задач з `tasks/` до `mt/` за специфікацією @7n/mt

**Status:** Accepted
**Date:** 2026-06-08

## Context and Problem Statement

Проект мав директорію `tasks/` з файлами `task.md`, що не відповідали специфікації пакету `@7n/mt`: відсутнє поле `schema_version: 1`, невідповідна директорія, залежності у frontmatter замість підкаталогу `deps/`.

## Considered Options

- Мігрувати `tasks/` → `mt/` з виправленням структури за специфікацією `@7n/mt`
- Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "Мігрувати `tasks/` → `mt/`", because специфікація пакету явно вимагає директорію `mt/` і поле `schema_version: 1`; обидва існуючих `task.md` були відсутні цих полів і знаходились у невідповідній директорії.

### Consequences

- Good, because задачі `mt/open-in-editor/task.md` і `mt/open-in-editor-local/task.md` тепер відповідають схемі: поле `schema_version: 1` додано, файли розміщені в `mt/`, залежність `open-in-editor-local → open-in-editor` зафіксована в `mt/open-in-editor-local/deps/open-in-editor.md`.
- Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- Специфікація: `node_modules/@7n/mt/docs/mt.md`
- Довідковий документ: `docs/mt.md`
- Конфіг: `.mt.json` з полями `mt_dir: "./mt"`, `worktrees_dir: "./.worktrees"`, `agent_concurrency: 5`, `default_budget_sec: 3600`, `default_budget_hard_sec: 10800`
- Видалена стара директорія `tasks/`
- Залежність `open-in-editor-local` від `open-in-editor` задокументована в `mt/open-in-editor-local/deps/open-in-editor.md`

## Update 2026-06-08

Відоме обмеження поточної реалізації: `find_tasks_dir` піднімається вгору по дереву і зупиняється на першому `.mt.json`/`mt/`; `scan_tasks` приймає один шлях. У монорепо з кількома workspace-ами (наприклад `api/mt/` та `task/mt/`) це повертає лише одну директорію. Три варіанти усунення обговорювалися, але рішення не прийнято на момент запису:

1. Агрегація через батьківський `.mt.json` з полем `workspaces`
2. Авто-discovery від кореня монорепо
3. Ручний multipath у UI (через `;`)
