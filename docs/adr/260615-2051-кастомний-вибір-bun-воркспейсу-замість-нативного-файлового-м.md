## ADR Кастомний вибір bun-воркспейсу замість нативного файлового менеджера

## Context and Problem Statement

У діалозі "Нова задача" (`CreateTaskDialog.vue`) для вибору проєктної директорії використовувався нативний файловий менеджер ОС (`open()` з `@tauri-apps/plugin-dialog`). Специфікація MT визначає, що `mt/` директорії можуть існувати лише всередині bun workspace директорій під `www/`, тому показ усієї файлової системи є надлишковим та неточним.

## Considered Options

* Залишити нативний файловий менеджер ОС (`open()`)
* Кастомний `q-select` з `list_project_workspaces` Tauri-командою, що сканує лише bun workspaces під `www/`

## Decision Outcome

Chosen option: "Кастомний `q-select` з `list_project_workspaces`", because MT-специфікація обмежує місця для `mt/` виключно bun workspace директоріями, тому показ лише цих директорій відповідає контракту і спрощує UX.

### Consequences

* Good, because transcript фіксує очікувану користь: користувач бачить лише валідні bun workspace директорії під `www/`, що унеможливлює вибір некоректного шляху.
* Good, because реалізовано multi-term fuzzy search — пробіл між словами ("abie k8s") розбивається на окремі токени, кожен з яких має зустрічатися в label; порядок не важливий.
* Bad, because якщо структура проєкту нестандартна (відсутній `package.json` або поле `workspaces`), директорія не з'явиться у списку.

## More Information

* Новий Tauri-команд `list_project_workspaces(root)` у `app/src-tauri/src/lib.rs` — сканує `www/<org>/<project>/`, читає `package.json`, розгортає `workspaces`-масив (підтримує glob `packages/*`), повертає `Vec<{path, label}>`.
* Новий Tauri-команд `home_dir()` у `app/src-tauri/src/lib.rs` — повертає `$HOME` через `std::env`, щоб уникнути хардкоду шляху `/Users/vitalii/www/`.
* Composable `app/src/composables/use-project-workspaces.js` — кешує результат виклику `list_project_workspaces`, має `refresh()` для примусового перезавантаження.
* `app/src/composables/use-projects-dir.js` змінено: `DEFAULT_PROJECTS_DIR` більше не хардкодить `/Users/vitalii/www/`; замість цього при першому запуску викликається `home_dir` і шлях будується динамічно.
* `filteredWorkspaces` у `CreateTaskDialog.vue` реалізовано як `computed`, а не `ref`, щоб автоматично синхронізуватися при завантаженні `workspaces`.

## ADR Кастомна SVG-іконка застосунку замість дефолтного білого квадрату Tauri

## Context and Problem Statement

Tauri-застосунок `task` містив дефолтну білу заглушку як іконку (у `src-tauri/icons/`). Для нормального відображення в доці та системних панелях потрібна власна іконка, що відповідає темі застосунку.

## Considered Options

* Використати зовнішнє зображення надане користувачем
* Згенерувати мінімалістичну SVG-іконку в стилі застосунку через `tauri icon` CLI

## Decision Outcome

Chosen option: "Згенерувати мінімалістичну SVG-іконку", because користувач явно обрав цей варіант — готового зображення не було.

### Consequences

* Good, because `tauri icon src-tauri/icons/icon.svg` автоматично згенерував усі необхідні формати: AppX `.png` різних розмірів, `icon.icns`, `icon.ico`, `128x128.png`, `128x128@2x.png`, `32x32.png`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

* SVG-іконка збережена в `app/src-tauri/icons/icon.svg` — темний фон (`#1e2a3a` → `#243447` градієнт), три рядки задач (два відмічені синіми галочками, один з `+`).
* Команда генерації: `cd app && bun run tauri icon src-tauri/icons/icon.svg`.
* Конфіг `tauri.conf.json` рядок 27 — `"icon"` масив вказує на `icons/32x32.png`, `icons/128x128.png`, `icons/128x128@2x.png`, `icons/icon.icns`, `icons/icon.ico`.
