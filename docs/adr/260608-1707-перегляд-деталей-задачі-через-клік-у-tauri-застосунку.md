## ADR Розповсюдження спільного markdown-документа через npm-пакет `@7n/mt`

## Context and Problem Statement
Файл дизайн-документа (`думка.MD` / `mt.md`) потрібен у двох проектах (`nitra/cursor` і `nitra/task`), але зберігатися має в одному місці. Symlink з абсолютним шляхом не підходить через перенесення на інші машини.

## Considered Options
* Абсолютний symlink (`ln -s /Users/vitaliytv/.../думка.MD`)
* Відносний symlink (`ln -s ../../cursor/docs/думка.MD`)
* Локальний пакет через `file:` (`"@nitra/cursor": "file:../cursor/npm"`)
* Опублікований npm-пакет `@7n/mt` з файлом у `docs/`

## Decision Outcome
Chosen option: "Опублікований npm-пакет `@7n/mt`, `docs/mt.md` у source пакету `mt/npm/docs/`", because `docs/` вже включена в поле `files` пакету; після публікації нової версії файл стабільно доступний на будь-якому комп'ютері через `bun install` без відносних шляхів між репозиторіями.

### Consequences
* Good, because на будь-якій машині достатньо `bun install` — `node_modules/@7n/mt/docs/mt.md` з'являється автоматично.
* Bad, because зміни у `mt.md` вимагають публікації нової версії пакету щоб `task` їх побачив; для активної локальної розробки рекомендовано `"@7n/mt": "file:../mt/npm"` як тимчасовий override.

## More Information
- Source файлу: `/Users/vitaliytv/www/nitra/mt/npm/docs/mt.md`
- `files` в `mt/npm/package.json` вже містить `"docs"`
- Поточна версія пакету: `0.1.0`; наступний крок — bump версії та `npm publish`
- Залежність у `task/package.json`: `"@7n/mt": "^0.1.0"`
- Lockfile підтверджує: пакет опублікований на npm (sha512 присутній)
