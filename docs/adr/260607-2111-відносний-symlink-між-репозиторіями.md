# Відносний symlink між репозиторіями cursor і task

**Status:** Accepted
**Date:** 2026-06-07

## Context and Problem Statement

Файл `cursor/docs/думка.MD` потрібен у двох git-репозиторіях (`nitra/cursor` і `nitra/task`), але зберігатися може лише в одному. Абсолютний symlink не є портативним між машинами.

## Considered Options

* Відносний symlink (відносний шлях між двома репо-сусідами)
* Абсолютний symlink
* Окреме shared-репо

## Decision Outcome

Chosen option: "Відносний symlink `../../cursor/docs/думка.MD`", because обидва репо (`cursor` і `task`) завжди є сусідами під `nitra/`, тому відносний шлях стабільний на будь-якій машині де дотримується ця структура.

### Consequences

* Good, because git зберігає symlink як текстовий рядок з відносним шляхом — він правильно клонується і відпрацьовує на новому комп'ютері якщо обидва репо в одній батьківській папці.
* Bad, because при клонуванні на новому комп'ютері обидва репо мають бути склоновані в одну батьківську директорію — інакше symlink буде битим.

## More Information

- Symlink: `task/docs/думка.MD -> ../../cursor/docs/думка.MD`
- Команда: `ln -s ../../cursor/docs/думка.MD /Users/vitaliytv/www/nitra/task/docs/думка.MD`
- Вимога: обидва репо клонувати в спільну батьківську папку (наприклад `~/www/nitra/`)

## Update 2026-06-07

- Claude Code не підтримує `@`-шляхи за межі git-репозиторію — спроба `@../cursor/docs/думка.MD` повертає `Failed to read file: ../cursor/docs/думка.MD`.
- Для підключення в `task/CLAUDE.md` використовується `@docs/думка.MD` — резолвиться через symlink до реального файлу в `cursor/docs/`.
- Варіанти відносного `@`-імпорту поза репо, npm-пакет та `file:` залежність розглядалися й відхилені як непрактичні для часто редагованого документа.
