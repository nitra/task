## ADR Перегляд деталей задачі через клік на рядок

## Context and Problem Statement
Застосунок `task` відображав граф задач у вигляді списку, але не мав способу переглянути вміст окремої задачі. Користувач не міг дізнатися деталі задачі (вміст `task.md`) безпосередньо з UI, не відкриваючи файли в редакторі.

## Considered Options
* Drawer-панель праворуч (side drawer)
* Модальний діалог по кліку на рядок
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Модальний діалог по кліку на рядок", because transcript зафіксував безпосередній перехід від drawer до `q-dialog` після того як `maximized`-drawer займав увесь екран; діалог обрано для обмеженого розміру (640px wide, 80vh max).

### Consequences
* Good, because transcript фіксує очікувану користь: функціонал запрацював у рідному Tauri-вікні після перебудови Rust-бінарника.
* Bad, because `invoke` з `@tauri-apps/api/core` не працює у звичайному браузері — тестування UI можливе лише у рідному Tauri-вікні.

## More Information
- Змінені файли: `app/src/components/TaskGraph.vue`, `app/src/components/TaskNodeItem.vue`, `app/src-tauri/src/lib.rs`
- Нова Tauri-команда: `read_task(tasks_dir: String, task_path: String) -> Result<String, String>` — читає `task.md` за шляхом вузла
- Встановлена залежність: `marked@18.0.5` для рендерингу markdown
- Клік на `task-row` емітує подію `select` вгору до `TaskGraph.vue`; батько відкриває `q-dialog` і асинхронно завантажує вміст через `invoke('read_task', ...)`
- Symlink: `task/docs/думка.MD → cursor/docs/думка.MD` для спільного дизайн-документу між двома проектами
