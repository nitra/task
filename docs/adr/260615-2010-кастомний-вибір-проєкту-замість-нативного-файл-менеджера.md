## ADR Кастомний вибір проєкту замість нативного файл-менеджера

## Context and Problem Statement
У діалозі «Нова задача» (`CreateTaskDialog.vue`) для вибору директорії проєкту використовувався нативний OS-файл-менеджер (`open()` з `@tauri-apps/plugin-dialog`), що дозволяв обрати будь-яку директорію на диску. Проте згідно зі специфікацією MT, директорія `mt/` може існувати лише всередині bun workspace. Нативний picker не обмежував вибір і не допомагав користувачу знайти лише валідні шляхи.

## Considered Options
* Замінити нативний файл-менеджер на кастомний `q-select` з пошуком, де список формується з bun workspaces
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Кастомний `q-select` з bun workspaces", because специфікація MT допускає `mt/` лише в bun workspace директоріях, тому список вибору обмежується саме ними, що усуває клас помилок вибору неправильної директорії.

### Consequences
* Good, because transcript фіксує очікувану користь: список показує лише валідні шляхи згідно зі специфікацією MT, без можливості обрати випадкову системну директорію.
* Bad, because якщо проєкт не має `workspaces` у `package.json` або використовує glob-паттерни, а не явний список (наприклад `"workspaces": ["packages/*"]`), він може не з'явитись у списку без додаткової обробки glob.

## More Information
- **Новий Tauri-команд:** `list_project_workspaces(root)` у `app/src-tauri/src/lib.rs` — сканує `<root>/*/package.json`, витягує `workspaces`-поля, повертає масив `{path, label}` (label у форматі `org / project / workspace`)
- **Новий composable:** `app/src/composables/use-project-workspaces.js` — кешує результат після першого виклику `invoke`
- **Оновлений компонент:** `app/src/components/CreateTaskDialog.vue` — `q-select` з `use-input` (пошук по label) замість кнопки `browse` + виклику `open()`
- Видалено залежність від `@tauri-apps/plugin-dialog` у `CreateTaskDialog.vue`
- `www/` як корінь читається з наявного `useProjectsDir()` composable
