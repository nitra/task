## ADR mt-scanner як окремий Rust crate з git-залежністю

## Context and Problem Statement
Логіка сканера була вбудована у Tauri-застосунок. Потрібно зробити її частиною `@7n/mt` (замінити JS-реалізацію), а також дозволити підключення до сторонніх проектів без залежності від локального шляху.

## Considered Options
* Публікація на crates.io
* Git-залежність (SSH URL)
* Cargo workspace (якщо обидва репо в одному монорепо)

## Decision Outcome
Chosen option: "Git-залежність (SSH URL)", because репо `nitra/mt` вже доступне через SSH і не потребує ручного публікування при кожній зміні.

### Consequences
* Good, because transcript фіксує очікувану користь: `cargo build` підтягує crate напряму з репо, Tauri `lib.rs` скорочено зі ~498 до ~44 рядків тонких wrapper-функцій; CLI-бінарник `mt-scanner scan <dir>` і `mt-scanner workspaces [<dir>]` виводить JSON.
* Bad, because `subdir` не підтримується у Cargo 1.95 (попередження "unused manifest key"), тому довелося додати кореневий `Cargo.toml` workspace у репо `nitra/mt`.

## More Information
Новий crate: `mt/scanner/` (`Cargo.toml`, `src/lib.rs`, `src/main.rs`). Workspace root: `mt/Cargo.toml` (`members = ["scanner"]`). Залежність у Tauri: `mt-scanner = { git = "ssh://git@github.com/nitra/mt.git" }`. Конфіг: `task/.cargo/config.toml` (`net.git-fetch-with-cli = true`). Commit `97e9f7f` у `nitra/mt`.
