## ADR Делегування всіх ФС-операцій у Rust (write-side)

## Context and Problem Statement
Tauri-застосунок `task` був read-only: Rust-крейт `mt-scanner` сканував файлову систему, JS-фронтенд лише відображав граф. Виникла потреба додати механізм **створення** нових задач безпосередньо з UI, що вперше вводить write-операцію і вимагає рішення, де розмістити логіку запису.

## Considered Options
* Rust-крейт `mt-scanner` (єдине джерело ФС-операцій)
* Node.js sidecar з `mt init` CLI
* Гібрид: JS для запису + Rust для читання

## Decision Outcome
Chosen option: "Rust-крейт `mt-scanner`", because директива користувача стверджує: «усе, що стосується роботи з файловою системою, має бути в Rust»; крейт уже є єдиним джерелом для читання, логічно тримати там і запис.

### Consequences
* Good, because write і read ідуть з одного контракту (`mt-scanner`), три споживачі (npm, bin, Tauri) лінкують один і той самий код.
* Bad, because transcript не містить підтверджених негативних наслідків.
