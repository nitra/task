## [0.8.0] - 2026-07-15

### Changed

- Додано новий icon.svg для owner

## [0.7.2] - 2026-07-15

### Changed

- `stryker-vue-macros-ignorer.mjs` — реекспорт спільної реалізації з `app/` замість повної копії (дедуплікація plugin-коду між пакетами).

## [0.7.1] - 2026-07-14

### Changed

- Перегенеровано файлову документацію OnboardingDialog.vue (побічний ефект прогону doc-files, без змін поведінки).

## [0.7.0] - 2026-07-12

### Added

- Посилання на ілюстрований гід для новачка у вступному діалозі («?»): «Повний ілюстрований гід» відкриває матеріал у системному браузері (де власник уже залогінений) — додано `tauri-plugin-opener`, якого раніше в owner не було.

## [0.6.0] - 2026-07-12

### Added

- Штаб (перша ітерація): лінива кнопка «бриф» на будь-якому рішенні черги — контекст, варіанти, рекомендація, наслідок відмови (omlx, JSON-схема). Анти-rubber-stamping: approve плану тепер відкриває діалог із найсильнішим запереченням, яке зміг згенерувати штаб (роль адвоката диявола) — власник має явно натиснути «я бачив заперечення — approve»; недоступність моделі не блокує рішення назавжди (є «продовжити без перевірки»). Спільний mission.js виніс дублювання витягання місії вузла з критика.

## [0.5.0] - 2026-07-12

### Added

- M3 — матриця автономії (перша ітерація, дані й редактор без runtime-виконавця): `autonomy.yml` — власний файл вузла (сусідній із `task.md`, поза контрактом mt-core — `a.md` перезаписується цілком при зміні виконавця), політика «клас дії → auto/approve» з ratchet-успадкуванням (дитина лише посилює, невідомий клас → fail-closed approve). Редактор у «Новій цілі» (запис при виборі альтернативи), read-only показ ефективної політики предків у картці plan-review. Tools `read_autonomy`/`write_autonomy`, валідація fail-closed на запис.

## [0.4.0] - 2026-07-12

### Added

- Онбординг першого запуску: діалог «Owner — пульт рішень власника» (що таке черга рішень, три режими, кнопки плановика/критика) з редактором project paths (tools `project_paths`/`set_project_paths` — раніше налаштувати шляхи з GUI було неможливо) і конфігом omlx-моделі; показується до першого «почати», далі доступний кнопкою «?» у тулбарі та CTA «налаштувати шляхи пошуку» з порожньої карти.

## [0.3.1] - 2026-07-12

### Changed

- release: app@0.5.4, owner@0.2.1

## [0.3.0] - 2026-07-12

### Added

- M2 — критик (перша ітерація): детермінований прогін після кожного rescan (мертві/неіснуючі залежності, цикли взаємного очікування, прострочені дедлайни, гілки без прогресу >7 днів включно з непризначеними) + семантичний LLM-прогін на вимогу (дублювання, суперечності цілей; по одному виклику omlx на воркспейс). Вердикти — картки у черзі рішень з явним «відхилити заперечення»; межа ретро-принципу: критик аналізує граф, не людей.

## [0.2.0] - 2026-07-11
