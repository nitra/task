## Огляд

`TOOLS` задає read-only каталог owner-дій для task graph. Секція потрібна, щоб мати єдине джерело доступних операцій без зміни стану чи запису даних.

## Поведінка

1. `TOOLS` задає єдиний каталог дій власника для роботи з task graph, щоб UI, оркестратор і LLM бачили однакову tool-поверхню.

2. `TOOLS` розділяє дії за рівнем довіри: читання доступне для безпечного огляду стану, а зміни призначені для рішень власника щодо планів і завершення робіт.

3. `TOOLS` описує виявлення робочих просторів і читання структури задач як read-only операції, щоб можна було аналізувати стан без зміни файлової системи чи бази даних.

4. `TOOLS` описує перегляд плану composite-задачі як read-only операцію, щоб власник міг оцінити дочірні роботи та поточний стан рішення перед дією.

5. `TOOLS` описує затвердження плану як зміну стану, що фіксує рішення власника й створює дочірні задачі після перевірки плану.

6. `TOOLS` описує відхилення плану як зміну стану з обов’язковою причиною, щоб рішення було зафіксоване й придатне для подальшого перегляду.

7. `TOOLS` описує прийняття роботи як зміну стану, що фіксує людське підтвердження завершення та підсумок досягнутого.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
