Компонент показує одну decision-картку з контекстом workspace, поточним станом, headline та, за потреби, переліком підзадач плану. Він дає змогу переглянути план, схвалити його, відхилити з причиною або прийняти вузол як виконаний із summary.

## Поведінка

1. Відображає базові дані рішення: стан, шлях вузла, назву workspace та короткий опис.
2. Обирає колір бейджа за рівнем stake, щоб швидко передати вагу рішення.
3. Показує блок плану лише тоді, коли вже є підзадачі для перегляду.
4. Дає доступ тільки до тих дій, які присутні в списку дозволених дій для цього рішення.
5. Для схвалення без уже завантаженого плану спершу підтягує children плану, щоб користувач бачив, що саме буде матеріалізовано.
6. Для approve запускає дію схвалення та після успіху повідомляє батьківський екран, щоб оновити дані.
7. Для reject відкриває діалог із причиною, не дозволяє відправити порожній текст і передає введену причину в дію відхилення.
8. Для done відкриває діалог із summary, не дозволяє відправити порожній текст і передає підтвердження виконання в дію прийняття.
9. Під час виконання будь-якої дії показує стан завантаження саме для відповідної кнопки.
10. Якщо серверна дія завершується помилкою, показує повідомлення про проблему через notification.

## Гарантії поведінки

- Дія не запускається, якщо вона не дозволена для цього рішення.
- Користувач не може відправити reject або done без заповненого тексту.
- Після успішної серверної дії картка сигналізує про потребу оновлення через подію.
- Помилки виконання не ламають інтерфейс, а повідомляються явно.
- План не показується, доки його підзадачі не були отримані або вже не відомі.
