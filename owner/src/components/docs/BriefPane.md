Компонент показує короткий бриф для власника: що змінилось із минулого візиту та які задачі чекають саме на нього. Він підкреслює не весь стан системи, а лише корисні для швидкого огляду зміни й особисті призначення.

## Поведінка

1. Якщо є зміни з минулого візиту, показує секцію з переліком дельт.
2. Якщо змін немає, показує порожній стан із повідомленням про відсутність змін.
3. Для кожної зміни відображає:
   - іконку та колір, що відповідають типу події;
   - шлях вузла;
   - людський опис самої зміни;
   - коротку назву workspace.
4. Якщо тип зміни невідомий, використовує нейтральне позначення.
5. Якщо зміна означає зникнення вузла, описує попередній стан окремо.
6. Якщо є персональні задачі, показує окрему секцію з переліком таких задач.
7. Для кожної персональної задачі відображає:
   - іконку користувача;
   - шлях вузла;
   - підпис, що задача чекає на виконавця;
   - назву workspace.
8. Якщо персональних задач немає, цю секцію не показує.

## Гарантії поведінки

- Бриф завжди розділяє загальні зміни та особисті задачі.
- Порожній стан для змін показується лише тоді, коли змін справді немає.
- Кожен рядок дельти має стабільне візуальне кодування за типом події.
- Невідомі або нові типи змін не ламають відображення.
- Особисті задачі завжди маркуються як такі, що очікують саме на виконавця.
- Відображення залишається лаконічним і придатним для швидкого сканування.
