## Огляд

Цей компонент відображає діалогове вікно для першого запуску, яке пояснює призначення "пульту рішень власника". Він збирає необхідні налаштування для роботи, такі як шляхи до проєктів та параметри LLM-моделі.

## Поведінка

1. Компонент ініціалізується з `modelValue` (boolean).
2. При першому відкритті (коли `modelValue` стає `true`), компонент викликає диспетчер для отримання поточної списку шляхів до проєктів (`project_paths`).
3. Користувач може додавати нові шляхи до проєкту через поле вводу та кнопку, або видаляти існуючі шляхи.
4. Користувач може налаштувати параметри LLM-моделі, такі як `baseUrl`, `model` та `apiKey`.
5. Коли користувач натискає кнопку "почати", компонент виконує такі дії:
   - Зберігає поточний список шляхів до проєктів, якщо він було змінено (`dirty` = true), викликаючи `set_project_paths`.
   - Позначає систему як пройдену онбординг (`markOnboarded`).
   - Закриває діалогове вікно (`emit`).
   - Повідомляє батьківський компонент, що онбординг завершено (`emit`).

## Гарантії поведінки

- Вікно відображає інструкції щодо того, що є "чергою рішень", а не звичайним дашбордом, описуючи три основних режими роботи: Рішення, Бриф та Карта.
- Шляхи до проєктів скануються для відображення у списку, і відображається повідомлення, якщо список порожній.
- Налаштування LLM-моделі зберігаються за допомогою `saveOmlx` при зміні полів.
- Якщо під час спроби зберегти шляхи виникає помилка, користувач отримує сповіщення, але процес не зупиняється повністю.
