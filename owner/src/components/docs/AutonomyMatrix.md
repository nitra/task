Компонент показує матрицю налаштувань autonomy для класів дій і дає змогу задати для кожного з них власний режим: успадкувати, auto або підпис. Він працює з редагуванням саме явної політики вузла, а не з ефективною політикою, що вже враховує предків.

## Поведінка

1. Для кожного класу дії та для загального правила відображається окремий рядок із назвою і трипозиційним перемикачем.
2. Поточний стан береться з переданої політики:
   - якщо для класу є явне значення, воно показується;
   - якщо значення відсутнє, вважається режим успадкування.
3. Перемикач дозволяє вибрати один із трьох станів:
   - успадкувати — прибрати явний запис для цього класу;
   - auto — дозволити автоматичну дію;
   - підпис — вимагати підтвердження.
4. Після зміни компонент надсилає оновлену політику назовні.
5. Підказка під матрицею пояснює, що успадкування означає відсутність власного вибору, а без рішення предка застосовується fail-closed поведінка.

## Гарантії поведінки

- Кожен клас дії має окремий незалежний перемикач.
- Режим успадкування не зберігається як явне значення.
- Відсутність ключа в політиці завжди означає успадкування.
- Зміна одного класу не змінює інші класи.
- Компонент не обчислює ефективну політику, а лише редагує власну явну.
