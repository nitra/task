Цей компонент відображає діалог для декомпозиції нової цілі. Він дозволяє користувачеві вказати ціль у природній мові, вибрати робочий простір та ID вузла, а потім згенерувати та обрати один із запропонованих планів.

## Поведінка

1. Користувач вводить необхідні дані: робочий простір, ID вузла цілі та опис цілі природною мовою.
2. Користувач може конфігурувати параметри моделі плановика (base URL, модель, API key) у розкривному елементі.
3. Після заповнення полів та натискання кнопки "згенерувати плани", компонент ініціює генерацію альтернативних планів на основі введеної цілі.
4. Система повертає до двох варіантів плану.
5. Користувач може переглянути кожен варіант, включаючи контекст, список завдань (з зазначенням залежностей) та потенційні ризики.
6. Після вибору одного із планів за допомогою кнопки "обрати цей план", компонент виконує дві асинхронні дії:
   - Створює вузол цілі з вказаним ID у вибраному робочому просторі.
   - Генерує чернетку плану, декомпозиції якої відповідає обраний варіант.
7. Після успішного виконання цих дій, відображається сповіщення, та діалог автоматично закривається.

## Гарантії поведінки

- Компонент вимагає, щоб був заданий робочий простір, ID вузла цілі (валідний формат) та текст цілі для ініціації генерації.
- При натисканні кнопки генерації, якщо результатів немає, він ініціює пошук та генерацію планів.
- При натисканні кнопки "обрати цей план", обидві операції (створення цілі та чернетки плану) виконуються атомарно. У разі помилки будь-якої з операцій, виникає сповіщення про помилку, а стан компонента повертається до попереднього.
