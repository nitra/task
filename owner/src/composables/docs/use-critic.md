## Огляд

`useCritic` надає безпечний доступ до критика: читає актуальний стан перевірок, а помилки перехоплює всередині, не пробиваючи їх назовні.

Це потрібно, щоб виклик критика не ламав роботу споживача навіть у разі збою, а результат залишався придатним для подальшого використання в UI та логіці оцінки.

## Поведінка

1. `useCritic` створює робочий стан критика для поточної сесії: список вердиктів і прапорець виконання.
2. Після кожного повного перегляду дерева він оновлює детерміновані вердикти, а семантичні вердикти власника зберігає в межах сесії окремо.
3. Для семантичної перевірки він готує стислий огляд воркспейсів і запускає аналіз по кожному воркспейсу незалежно, щоб не зупиняти весь прогін через локальну проблему.
4. Якщо якийсь воркспейс не вдалося прочитати або відповідь моделі недоступна, `useCritic` не перериває роботу й просто пропускає цей випадок.
5. Після семантичного прогону він поєднує нові вердикти з уже наявними детермінованими вердиктами, щоб зберегти актуальний стан критики.
6. Коли власник відхиляє вердикт, `useCritic` прибирає його з поточної сесії, щоб він не створював повторний шум.
7. Усі дії виконуються без запису у файлову систему чи базу даних і без пробросу помилок назовні.

## Публічний API

- useCritic — збирає вердикти критика та запускає два прогони: детермінований і семантичний

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
