## Огляд

Файл потрібен для `usePlanner`: він обирає LLM-конфігурацію власника для планування та повертає окремий результат або помилку для кожної спроби. Він працює fail-safe, тому помилки мережі чи моделі не виходять назовні у вигляді винятків і не ламають виклик.

## Поведінка

1. `usePlanner` надає стан для вибору LLM-конфігурації в межах власника: базову адресу, модель, API key та дію збереження налаштувань.
2. `usePlanner` запускає генерацію альтернатив плану для одного наміру через кілька незалежних напрямів аналізу паралельно.
3. `usePlanner` звертається до мережі для кожного напряму окремо, щоб отримати незалежний результат від моделі.
4. `usePlanner` повертає по кожному напряму або згенеровану альтернативу, або помилку, щоб користувач бачив повну картину навіть при частковому збої.
5. `usePlanner` не піднімає винятки назовні: помилки перетворює на безпечний результат для подальшого показу в інтерфейсі.

## Публічний API

- usePlanner — збирає налаштування моделі й створює варіанти декомпозиції задачі

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
