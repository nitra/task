## Огляд

Секція надає спільний read-only стан лісу задач через `useForest` для синхронного перегляду поточного стану між оновленнями сесії. Вона потрібна, щоб отримувати єдину точку доступу до даних лісу без внесення змін і без окремого кешування.

## Поведінка

1. useForest надає спільний стан лісу задач для owner-екранів: список воркспейсів, дерево вузлів по кожному воркспейсу, дельту відносно базового знімка та ознаку завантаження.

2. useForest під час оновлення збирає актуальний перелік воркспейсів, проходить по кожному з них і формує сукупний ліс задач для перегляду.

3. useForest порівнює новий стан із базовим знімком першого завантаження сесії, щоб зміни не «самозатиралися» під час подальших оновлень.

4. useForest фіксує зміни у вигляді дельти для owner-інтерфейсу, щоб швидко бачити, що змінилося від останнього візиту.

5. useForest реагує на зміни директорій задач через файлові події: після короткої паузи запускає повторне оновлення стану, якщо середовище підтримує такі події.

6. useForest у тестовому або не-Tauri середовищі працює без live-оновлень, але зберігає можливість ручного оновлення стану.

7. useForest не змінює дані у ФС чи БД; він лише читає актуальний стан і відображає його в пам’яті для UI.

## Публічний API

- useForest — дає owner-екранам спільний доступ до сховища задач лісу.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
