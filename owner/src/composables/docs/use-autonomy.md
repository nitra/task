## Огляд

Обчислює ефективну політику для вузла, щоб отримати єдине підсумкове правило для подальшого використання системою. Функція працює read-only і поводиться fail-safe: не кидає винятків назовні, навіть якщо під час обчислення виникає помилка.

## Поведінка

1. **effectivePolicyFor** визначає ефективну політику вузла як підсумок політик усіх його предків, рухаючись від кореня до самого вузла.
2. Для кожного предка бере його власну автономну політику, якщо вона є, і додає її до вже накопиченого результату.
3. Якщо для предка файл політики відсутній, цей крок вважається нейтральним: попередній результат не змінюється.
4. Якщо файл політики предка пошкоджений або не читається коректно, обчислення не зупиняється; вузол продовжує отримувати політику з решти ланцюжка.
5. Після обробки всіх предків повертає зведену ефективну політику для вузла.
6. Функція працює лише в режимі читання: нічого не змінює у файловій системі чи базі даних і не викидає помилки назовні.

## Публічний API

- effectivePolicyFor — Обчислює підсумкову політику вузла, послідовно враховуючи autonomy.yml усіх предків від кореня; якщо у предка немає файлу, цей крок не змінює результат.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
