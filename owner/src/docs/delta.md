## Огляд

`snapshotForest`, `diffForest`, `readSnapshot`, `writeSnapshot` працюють із знімками дерева вузлів між візитами: перші дві функції формують і порівнюють стан, а `readSnapshot` і `writeSnapshot` дають змогу отримати або зберегти знімок для наступного візиту. Модуль `read-only`, не пише у ФС чи БД, і працює fail-safe: перехоплює помилки та не кидає винятків назовні, а за окремих помилок повертає порожнє значення замість винятку.

## Поведінка

- snapshotForest — перетворює дерево вузлів на плаский знімок станів за шляхами для порівняння між візитами.
- diffForest — порівнює два знімки лісу й повертає впорядкований список змін із пріоритетом для ескалацій, завершень, появ і решти змін.
- readSnapshot — читає збережений знімок попереднього візиту зі сховища й повертає null, якщо дані недоступні або це перший візит.
- writeSnapshot — зберігає поточний знімок для наступного візиту й мовчки пропускає помилки сховища.

## Публічний API

- snapshotForest — Показує плоский знімок стану кожного вузла в лісі за його шляхом.
- diffForest — Порівнює два знімки лісу й дає рядки змін для брифу.
- readSnapshot — Читає збережений знімок минулого візиту; повертає null, якщо це перший візит або не браузер.
- writeSnapshot — Записує поточний знімок лісу, щоб використати його під час наступного візиту.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
