## Огляд

Модуль надає три публічні точки входу: `runDeterministicCritic`, `parseCriticReply`, `criticPrompt`. Він потрібен, щоб отримувати керований критичний висновок про стан дерева задач у workspace та окремо готувати й розбирати запит/відповідь для LLM без виходу помилок назовні. Модуль read-only і не пише у ФС чи БД; усі помилки обробляє fail-safe, щоб прогін не падав через збій відповіді або обробки.

## Поведінка

- `runDeterministicCritic` — детерміновано переглядає дерево задач у всіх workspace і повертає вердикти про прострочені дедлайни, залежності на мертві або відсутні вузли, застарілі гілки та цикли взаємного очікування; працює лише в межах переданого forest і не пише у ФС чи БД.
- `parseCriticReply` — безпечно розбирає сирий текст відповіді LLM у список вердиктів, відкидаючи невалідні елементи, щоб шум моделі не ламав прогін.
- `criticPrompt` — готує system/user-повідомлення для семантичного критика: задає фокус на вадах плану й структури та вимагає суворий JSON-масив без обгортки.

## Публічний API

- runDeterministicCritic — послідовно проганяє критичні перевірки по всьому лісу без випадковості, щоб отримати стабільний результат.
- parseCriticReply — витягує з відповіді LLM придатні вердикти, а все зайве або зламане тихо ігнорує, щоб шум моделі не ламав прогін.
- criticPrompt — задає текст запиту для семантичного критика в межах одного workspace.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
