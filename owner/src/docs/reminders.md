## Огляд

`deriveReminders`, `applySnoozes`, `bucketPersonal` і `reminderRibbon` разом формують особистий список нагадувань із deadline, застарілих ескалацій і персональних snooze. `ESCALATION_STALE_DAYS` визначає, коли ескалація вважається застарілою, а `nextMidnight` задає межу для добового оновлення. Це потрібно, щоб відокремити особисті нагадування від решти задач і показати їх у компактному вигляді через `reminderRibbon`.

## Поведінка

- `ESCALATION_STALE_DAYS` — задає поріг у днях, після якого відкрита ескалація на мою адресу вважається застарілою.
- `nextMidnight` — повертає момент наступної локальної півночі як стандартний час для відкладеного нагадування.
- `deriveReminders` — збирає активні нагадування з workspace-лісу за правилами deadline та застарілої ескалації, відсікаючи не-мій scope.
- `applySnoozes` — прибирає з переліку нагадувань ті, для яких ще діє персональний snooze.
- `bucketPersonal` — групує особисті задачі брифу в кошики дедлайнів від найтерміновішого до пізнішого.
- `reminderRibbon` — формує коротку стрічку-лічильник для режиму «Рішення» на основі активних нагадувань.

## Публічний API

- ESCALATION_STALE_DAYS — поріг давності, після якого ескалація вважається протермінованою
- nextMidnight — ставить дефолтне відкладення до наступної локальної півночі
- deriveReminders — витягає нагадування мого скоупу за трьома правилами спеки
- applySnoozes — ховає нагадування, які ще не вийшли з snooze
- bucketPersonal — розкладає особисті задачі брифу по кошиках дедлайнів від найтерміновішого; порожні не показує
- reminderRibbon — показує в режимі «Рішення» компактну стрічку лічильників нагадувань, не впливаючи на вибір режиму
