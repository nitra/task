## Огляд

Файл формує текстовий контур ухвалення рішення: зводить вузол і причину в черзі в короткий дайджест, а також працює з JSON-контрактами для брифу та заперечення через `briefingPrompt`, `parseBriefing`, `objectionPrompt`, `parseObjection`. Це потрібно, щоб перетворювати контекст рішення на структурований бриф або заперечення у погодженому форматі.

## Поведінка

- buildDecisionDigest — збирає лаконічний текстовий дайджест рішення з вузлом, причиною в черзі, контрактом і, за потреби, списком підзадач плану.
- briefingPrompt — формує промпт для брифу: просить дати контекст, варіанти, рекомендацію та наслідок відхилення у вигляді одного JSON-обʼєкта.
- parseBriefing — розбирає відповідь брифу, перевіряє наявність обовʼязкових полів і повертає очищений структурований бриф.
- objectionPrompt — формує промпт для анти-rubber-stamping: просить знайти найсильніше конкретне заперечення або прямо сказати, що вагомих заперечень не видно.
- parseObjection — розбирає відповідь із запереченням і повертає непорожній текст заперечення.

## Публічний API

- buildDecisionDigest — Збирає стислий текстовий дайджест рішення для LLM: чому задача потрапила в чергу, який контракт треба виконати, а для plan-review ще й перелік підзадач.
- briefingPrompt — Формує промпт для брифу з контекстом, варіантами, рекомендованою дією та наслідком відмови.
- parseBriefing — Розбирає відповідь брифу й зупиняється з поясненням, якщо бракує обов’язкових полів.
- objectionPrompt — Формує промпт для заперечення проти автоматичного погодження, щоб витягнути найсильніший контраргумент.
- parseObjection — Розбирає відповідь заперечення.
