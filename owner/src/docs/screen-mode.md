## Огляд

`chooseMode` визначає режим першого екрана для переданого стану. Функція потрібна, щоб централізувати вибір відображення і не записує дані у ФС/БД.

## Поведінка

1. `chooseMode` визначає пріоритетний стан першого екрана за фіксованим порядком: спочатку рішення, потім зміни, потім карта портфеля.

2. Якщо є хоча б одне рішення, `chooseMode` повертає режим рішень і формує заголовок із кількістю рішень, правильною українською формою слова та дієсловом очікування.

3. Якщо рішень немає, але є зміни з минулого візиту, `chooseMode` повертає режим короткого огляду й пояснює, що екран показує саме нові зміни.

4. Якщо немає ні рішень, ні змін, `chooseMode` повертає режим карти портфеля й пояснює, що це спокійний стан без актуальних дій.

5. `chooseMode` працює детерміновано: однаковий стан завжди дає однаковий режим і заголовок.

6. `chooseMode` лише обирає бізнес-режим і текст пояснення, не змінює зовнішній стан і не записує дані.

## Публічний API

- chooseMode — визначає режим стартового екрана та пояснювальний заголовок для нього.
