## Огляд

Цей модуль відповідає за відстеження стану первинного оформлення користувача (`isOnboarded`). Публічний інтерфейс дозволяє перевіряти цей стан. Для реєстрації факту проходження вступного діалогу використовується функція `markOnboarded`. Функції працюють у режимі fail-safe: вони не кидають винятків назовні і можуть повертати порожнє значення, наприклад, `null`, у разі внутрішніх помилок. Модуль не реалізує механізму кешування.

## Поведінка

isOnboarded повертає стан, чи було представлено вступне діалогове вікно користувачеві, враховуючи як збережені дані в браузері, так і стан поточної сесії.
markOnboarded фіксує, що користувач пройшов вступне діалогове вікно, зберігаючи цю інформацію для подальших сесій та поточної сесії.

## Публічний API

isOnboarded — вказує, чи завершив користувач процес первинного налаштування на поточному пристрої чи в поточній сесії.
markOnboarded — позначає факт успішного завершення процесу первинного налаштування.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
