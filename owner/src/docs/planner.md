## Огляд

Файл задає контракт для роботи плановика з лінзами декомпозиції, формуванням prompt-інструкції, перевіркою node-id і перетворенням відповіді моделі в деревоподібну структуру. `LENSES` задає набір лінз, `plannerPrompt` — текст завдання для моделі, `validNodeId` — допустимість node-id, `parseAlternative` — інтерпретацію альтернативи, `childrenToYaml` — подання дочірніх елементів у YAML-форматі.

## Поведінка

- LENSES — містить дві альтернативні лінзи декомпозиції цілі: для мінімального шляху та для повного покриття ризиків.
- plannerPrompt — формує пару повідомлень для плановика, щоб LLM згенерувала план під задану лінзу в заданому структурованому форматі.
- validNodeId — перевіряє, чи є id валідним node-id у форматі kebab-case латиницею без слешів.
- parseAlternative — розбирає сирий текст відповіді моделі в альтернативу плану, відкидаючи невалідну форму й перевіряючи наявність підзадач.
- childrenToYaml — серіалізує підзадачі у YAML-блок children для plan-файлу з тим самим форматом, який читає parser плану.

## Публічний API

- LENSES: Список лінз для побудови плану й розбиття роботи на окремі напрямки.
- plannerPrompt: Формує запит до плановика для однієї лінзи.
- validNodeId: Переконується, що node-id не порожній, записаний латиницею в kebab-case і не містить «/».
- parseAlternative: Перетворює відповідь моделі на альтернативу плану; відкидає невалідну форму й повертає зрозумілу причину, а ` ```json``` `-обгортки приймає.
- childrenToYaml: Записує підзадачі в YAML-блок `children:` у форматі `plan_NNN.md`, сумісному з `mt-core parse_children`.
