## Огляд

Запускає read-only build-підготовку застосунку без запису у файлову систему чи базу даних. Файл існує, щоб перевірка перед компіляцією не змінювала дані застосунку.

## Поведінка

1. Запускає стандартну підготовку Tauri-застосунку на етапі збирання.
2. Делегує формування build-артефактів механізму Tauri, щоб застосунок отримав потрібні метадані та ресурси для подальшої компіляції.
3. Не змінює дані застосунку у файловій системі чи базі даних під час виконання власної логіки.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
