## Огляд

Система надає механізм для роботи з журналами, який здійснюється через окремий бінарний процес. Він знаходить необхідний бінарний файл журналу, використовуючи змінні середовища або шляхи збірки, і надає інтерфейс для взаємодії з записами. Усі операції з журналами виконуються синхронно, при цьому контекст передається до процесу.

## Поведінка

1. Для ініціалізації сховища журналів для Node.js, використовуючи `createNodeJournalStore`, система знаходить виконуваний файл бінарного журналу. Якщо змінна середовища `TASK_JOURNAL_BIN` встановлена, використовується її шлях. Інакше, система шукає бінарний файл у директоріях збірки Rust-проекту. Якщо бінарний файл не знайдено, виникає помилка.
2. Після знаходження бінарного файлу, система створює сховище, яке надає три основні методи для взаємодії з журналом:
    а. Для створення нового запису викликається метод `create` з об'єктами `intent` та `actor`, що призводить до ініціалізації запису у Rust-бінарному сховищі та повертає його унікальний ідентифікатор.
    б. Метод `load` приймає ідентифікатор і повертає повний об'єкт запису, що був збережений.
    в. Метод `update` приймає ідентифікатор та патч-об'єкт, що застосовує зміни до відповідного запису у сховищі.
3. Під час виконання будь-якої операції, що взаємодіє з журналом, система передає поточні змінні середовища у Rust-бінарний процес, включаючи можливий перехоплення шляху запитів через `TASK_REQUESTS_DIR`.
4. Взаємодія з Rust-бінарним процесом відбувається синхронно, і якщо процес завершується з ненульовим кодом статусу, виникає виняток.
