## Огляд

Файл визначає візуальні параметри для різних станів завдань у системі, забезпечуючи повний набір даних (іконки, акцентні кольори, мітки) для рендерингу кожного стану. Він використовує константи `STATE_CONFIG` та `stateConfig` для визначення цих параметрів. У разі відсутності специфічної конфігурації, він застосовує стандартний візуальний набір, беручи дані з `FALLBACK_STATE`.

## Поведінка

STATE_CONFIG — надає централізовану конфігурацію для візуального представлення станів завдань (іконка, акцентний колір, мітка), забезпечуючи консистентність вигляду в різних частинах візуалізації.
FALLBACK_STATE — визначає стандартний візуальний набір для випадків, коли конкретний стан завдань не може бути знайдений у конфігурації.
stateConfig — повертає візуальну конфігурацію для заданого стану завдань, використовуючи `waiting` у випадку відсутності відповідного запису.

## Публічний API

STATE_CONFIG — Визначає візуальні налаштування для певного стану завдання.
FALLBACK_STATE — Стандартний стан, який використовується у разі відсутності іншої конфігурації.
stateConfig — Визначає візуальний вигляд стану завдання, якщо інша конфігурація недоступна, використовується стан очікування.
