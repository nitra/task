## Огляд

Цей файл відповідає за збір (агрегацію) вузлів, які вимагають ручного втручання, з усіх доступних робочих просторів. Він аналізує дерево вузлів, щоб визначити причини та пріоритети кожного вузла на основі його поточного стану та налаштувань, заданих у конфігурації `ATTENTION`. Функція формує та презентує чіткий, терміновий список вузлів, які користувачеві необхідно розглянути.

Збирає вузли з усіх воркспейсів, які потребують втручання людини, аналізуючи дерева вузлів за шляхами воркспейсів.
Для кожного зібраного вузла визначає причину та пріоритет шляхом порівняння його поточного стану зі значеннями, визначеними у конфігурації `ATTENTION`.
Сортує зібрані вузли у порядку терміновості; менший номер пріоритету відповідає вищій терміновості.
Повертає відсортований список вузлів, що потребують уваги, включаючи дані про вузол, відповідний воркспейс та причину його виявлення.

## Поведінка

1. Збирає вузли з усіх воркспейсів, які потребують втручання людини, аналізуючи дерева вузлів за шляхами воркспейсів.
2. Визначає причину та пріоритет для кожного зібраного вузла на основі його поточного стану, порівнюючи його зі значеннями у конфігурації `ATTENTION`.
3. Сортує зібрані вузли у порядку терміновості, де менший номер пріоритету означає вищу терміновість.
4. Повертає відсортований список вузлів, що потребують уваги, включаючи інформацію про вузол, відповідний воркспейс та причину його потрапляння до списку очікування.

## Публічний API

collectAttention — збирає усі вузли, які вимагають втручання людини, рекурсивно шукаючи їх у всіх підструктурах воркспейсів та сортуючи зібрані вузли за їхньою терміновістю.
