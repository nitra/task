## Огляд

Цей модуль забезпечує валідацію вхідних даних для створення завдань, визначаючи конфігураційні шляхи на основі `.mt.json` та формуючи структуровані об'єкти параметрів.

Поведінка:
validateTaskName перевіряє коректність назви завдання згідно з правилами, надаючи повідомлення про помилку, якщо назва не відповідає вимогам.
mtDirFor визначає та нормалізує шлях до папки `mt` у кореневому каталозі проєкту.
buildCreateOpts формує об'єкт опцій для ініціалізації завдання, включаючи лише надані поля та виключаючи ті, що специфічні для агента для людського режиму.

## Поведінка

Поведінка:
validateTaskName перевіряє коректність назви завдання відповідно до правил і видає повідомлення про помилку у читабельному форматі, якщо назва не валідна.
mtDirFor визначає шлях до папки `mt` у кореневому каталозі проєкту, нормалізуючи його.
buildCreateOpts формує об'єкт опцій для створення завдання, відбираючи лише ті поля, що були надані, та виключаючи поля, специфічні для агента, для людського режиму.

## Публічний API

- validateTaskName — Перевіряє коректність ідентифікатора вузла завдання у стилі, прийнятому в Rust, до будь-яких операцій з файловою системою.
- mtDirFor — Визначає директорію завдань `mt/` для обраного кореневого каталогу проекту, використовуючи стандартне розміщення `.mt.json`.
- buildCreateOpts — Формує об'єкт `opts` у форматі snake\_case для створення завдання, виключаючи пусті поля, щоб забезпечити автоматичне використання значень за замовчуванням з `.mt.json` для всіх непередбачених полів.
