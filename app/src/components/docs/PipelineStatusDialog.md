## Огляд

Компонент відображає список конфігурованих CI/CD конвеєрів (pipelines) для робочого простору та надає можливість сортування даних. Він використовує `QDialog` для інтерактивного відображення результатів прогонів.

## Поведінка

1. **Ініціалізація та Завантаження**:
    * При відкритті діалогу ініціюється асинхронний виклик `invoke` для отримання поточних прогонів.
    * Поки відбувається завантаження, відображається спінер (`loading = true`).
    * У разі помилки виклик ловиться, і помилка відображається в індикаторі (`errorMessage`).
    * Після завершення завантаження стан `loading` встановлюється на `false`.
2. **Сортування**:
    * Існує механізм для перемикання режиму сортування між 'Date' та 'A-Z' за допомогою `q-btn-toggle`.
    * Сортування впливає на порядок відображення результатів (`sortedRuns`) відповідно до обраного режиму.
3. **Відображення Даних**:
    * Список прогонів відображається у вигляді таблиці (`q-markup-table`).
    * Кожна комірка відображає статус у вигляді пігулки (`status-pill`), яка використовує динамічне колірне значення з `pipelineStateConfig`.
    * Рядок, що завершився невдачею (`conclusion === 'failure'`), отримує клас `--clickable`, що дозволяє клікати на рядок для відкриття деталей.
4. **Перегляд Деталей**:
    * Клікабельність рядка активується, якщо `run.conclusion` дорівнює `'failure'`.
    * Клік на рядок викликає функцію `openRunDetails`, яка встановлює обраний запуск (`selectedRun`) та відкриває діалог деталей (`runDetailsOpen = true`).

## Гарантії поведінки

* Усі дані, отримані з API, використовуються для формування списку `runs`.
* Сортування даних виконується відповідно до обраного режиму (`sortMode`).
* Доступ до деталей прогону здійснюється лише для невдалих прогонів.
* Усі змінені стани (завантаження, помилки) коректно керуються через `ref` та `computed` властивості.
