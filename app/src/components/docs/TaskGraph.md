Компонент відображає візуальне дерево завдань (Task Graph) для різних робочих просторів (workspaces). Він надає панель для керування, підсумковий звіт про стан завдань та деталі для вибору одного вузла графа.

## Поведінка

1. При ініціалізації компонента викликається функція `scanAll`, яка отримує список робочих просторів і сканує кожен із них, заповнюючи дані про вузли завдання.
2. Відстежуються зміни в файловій системі (через подію `mt-changed`); при їх отриманні запускається відкладений повторний сканування графа.
3. На панелі відображаються кнопки для дії: перегляд журналу (`auditOpen`), звернення до агента (`agentOpen`), створення нового завдання (`createOpen`) та оновлення графа (`scanAll`).
4. Відображається інформація про помилки (`error`), якщо процес сканування не вдався.
5. Відображається зведений підсумок (`stateCounts`), що показує кількість завдань у кожному стані.
6. Відображається розділ "Needs attention" (`attention`), який виділяє вузли, що вимагають уваги користувача.
7. Відображаються окремо для кожного робочого простору (`workspaces`) графічні елементи завдань (`TaskNodeItem`).
8. При кліку на вузол або його залежність (`onSelect`/`onSelectDep`) відкривається детальний діалог.
9. У детальному діалозі:
   - Відображається загальна інформація про вибране завдання та його стан.
   - Доступні дії для вузла (`NodeActions`).
   - Відображається "ланцюг артефактів" (`ArtifactChain`).
   - При виборі артефакта завантажується його вміст (`taskContent`) та відображається у вікні з Markdown-рендерингом.
10. При зміні стану вузла (`onNodeChanged`) або після створення нового завдання (`onCreated`), графа та стан вибраного вузла автоматично оновлюються шляхом повторного сканування.

## Гарантії поведінки

- Компонент гарантує, що дані про стан завдань та їх залежності відображаються з урахуванням актуальних даних після сканування.
- При відсутності робочих просторів або помилках, компонент відображає відповідні повідомлення.
- При виборі вузла, детальний перегляд містить вміст артефакту, який коректно парситься як Markdown.
- Обробляються як прямі виклики, так і зміни, спричинені зовнішніми подрами файлової системи.
