## Огляд

Компонент відображає поточний стан виконання роботи агента, зчитуючи чернетки з файлу `run-draft.md`. Він постійно оновлюється, щоб відображати останні статуси.

## Поведінка

1. При монтуванні компонента ініціюється періодичний процес опитування (polling) кожні 2500 мс.
2. На кожному циклі опитування компонент звертається до API для зчитування вмісту `run-draft.md` для заданої ноди.
3. Зчитаний текст обробляється та парситься, щоб отримати дані про статус.
4. В інтерфейсі відображається індикатор "Running" з анімацією, доки дані не будуть отримані або будуть відсутні.
5. Якщо дані доступні, відображаються окремі секції, які містять:
   - Поточний статус завершення (`Completed`).
   - Перелік блокувальних факторів (`Blockers`).
   - Інформацію про наступну спробу (`Next Attempt`).
6. Якщо статус не визначений, відображається повідомлення про відсутність нотаток у чернетки.
7. При зміні шляху ноди або каталогу завдань, компонент скидає поточний стан і ініціює нове опитування.

## Гарантії поведінки

- Компонент гарантовано періодично оновлює відображення статусу виконання.
- У випадку тимчасових помилок при зчитуванні файлу, компонент не зависне і спробує знову через наступний інтервал.
- Відображення буде коректно ініціалізовано, навіть якщо дані відсутні при першому завантаженні.
