## Огляд

Компонент відповідає за відображення діалогового вікна для створення нової задачі. Він збирає необхідні параметри, такі як проєкт, назва, налаштування агента та інші метадані.

## Поведінка

1. При відображенні діалогу, компонент ініціалізує поле назви та встановлює останньо вибраний проєкт.
2. Виконується завантаження списків директорій проєктів та робочих просторів для обраного проєкту.
3. Користувач може вибрати проєкт із попередньо сконфігуровазованого списку або додати новий шлях до пошуку проєкту.
4. Користувач вводить назву задачі, яка проходить валідацію.
5. Користувач обирає режим виконання: Автоматизований (Agent) або Ручний (Human).
6. Якщо обрано режим "Agent", користувач може вказати модель, обрати навички та визначити бюджет.
7. Користувач вводить підказку та може визначити залежності від інших ідентифікаторів вузлів.
8. При натисканні кнопки "Створити", компонент збирає всі дані та передає їх для виконання операції створення завдання через зовнішній диспетчер.
9. У відповідь на успішне створення, відображається повідомлення про створення, а також вимикається діалог.
10. При натисканні "Скасувати", діалог закривається без змін.

## Гарантії поведінки

- Створення завдання може бути ініційоване лише за умови наявності обраного проєкту та коректної назви задачі.
- Якщо операція створення успішна, компонент сповіщає про це та знімає фокус з діалогу.
- При відмові створення завдання (наприклад, якщо воно вже існує), користувач отримує відповідне сповіщення.
