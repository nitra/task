Компонент показує доступні дії для вузла задачі залежно від його стану та ролі виконавця. Він дає змогу переглядати план, призначати виконавця, надсилати human-сигнали, а також виконувати небезпечні операції над вузлом.

## Поведінка

- Для вузла на перегляді плану:
  - показує файл плану та список майбутніх дочірніх задач;
  - дозволяє підтвердити план і матеріалізувати дочірні вузли;
  - дозволяє відхилити план лише після введення причини.

- Для непризначеного вузла:
  - пропонує вибір між agent та human-виконавцем;
  - для agent дозволяє вибрати рівень моделі;
  - для human дозволяє вказати кваліфікацію;
  - після підтвердження призначає вибраного виконавця вузлу.

- Для активного human-вузла:
  - приймає короткий підсумок результату;
  - дозволяє завершити вузол як done або передати його на audit;
  - дозволяє позначити виконання як failed лише після заповнення виконаного, блокерів і рекомендації для наступної спроби.

- Для будь-якого вузла:
  - показує небезпечні дії invalidate та kill;
  - вимагає додаткове підтвердження перед виконанням небезпечної дії;
  - відображає пояснення наслідків вибраної небезпечної дії.

- Після успішної мутації повідомляє батьківський компонент, що граф змінився.
- Під час виконання дії блокує повторні запуски індикатором завантаження.
- У разі помилки показує текст помилки в блоці дій.

## Гарантії поведінки

- Дії плану доступні лише для вузлів у стані перегляду плану.
- Призначення виконавця доступне лише для непризначених вузлів.
- Human-сигнали доступні лише human-вузлам у робочих станах.
- Reject неможливо підтвердити без причини.
- Done та Audit неможливо надіслати без підсумку результату.
- Failed неможливо підтвердити без повної діагностики.
- Небезпечна дія не виконується одразу після першого натискання — потрібне підтвердження.
- Помилки дій не приховуються від користувача.
