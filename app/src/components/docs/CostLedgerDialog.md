Компонент відображає детальний звіт про витрати (Cost Ledger) для заданого робочого простору. Він отримує дані про виконання завдань та відображає загальні підсумки й деталі по кожному елементу.

## Поведінка

1. При відкритті діалогового вікна (коли властивість `modelValue` стає `true`), компонент ініціює запит до бекенду, використовуючи функцію `cost_ledger` з API. Запит передає шлях до директорії завдань (`tasksDir`).
2. Під час очікування відповіді від бекенду відображається індикатор завантаження.
3. Після отримання даних, компонент відображає загальні показники за всі виконані елементи: кількість прогонів, час виконання, загальну вартість та кількість токенів вхідних/вихідних.
4. Якщо дані про елементи (`ledger.nodes`) відсутні, відображається повідомлення про відсутність файлів.
5. Якщо даних про елементи є, компонент відображає таблицю, яка деталізує показники для кожного елемента (`entry.path`): кількість прогонів, час виконання, вартість та кількість токенів вхідних/вихідних.
6. При закритті діалогового вікна або зміні властивості `modelValue` на `false`, компонент зупиняє будь-який фоновий запит і зникає.

## Гарантії поведінки

- Компонент гарантовано відобразить помилку, якщо виклик `cost_ledger` поверне помилку.
- Відображення загального підсумку та таблиці відбувається лише після успішного отримання даних про леджер.
- Всі часові та вартісні значення форматуються перед відображенням для кращої читабельності.
