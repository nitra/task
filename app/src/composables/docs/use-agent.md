## Огляд

Про цей модуль здійснюється ініціалізація шлюзу агента, що забезпечує готовність середовища для взаємодії з моделлю. Модуль налаштовує контекст, дозволяючи агенту використовувати визначений набір інструментів та системний промпт, сформований на основі заданих робочих просторів. Для початку роботи з агентом використовується функція `useAgent`. Модуль повертає конфігурований об'єкт, що містить усі необхідні дані та інтерфейси для коректної роботи. Важливо зазначити, що взаємодія з цією частиною системи є read-only.

## Поведінка

1. Викликає функцію useAgent для ініціалізації шлюзу вбудованого агента в застосунку.
2. Шлюз отримує доступ до каталогу інструментів (`TOOLS`), який визначає доступні можливості агента.
3. Агент отримує системний промпт, згенерований на основі списку робочих просторів (`ctx.workspaces`).
4. Запит агента прив'язується до контексту робочих просторів, забезпечуючи його ґрунтування.
5. Конфігурується використання OMLX для роботи з моделлю, встановлюючи префікс сховища 'task' та модель за замовчуванням 'gemma-4-e4b-it-OptiQ-4bit'.
6. Повертає об'єкт, що містить посилання на базовий URL, модель, ключ API, журнал, а також методи для надсилання запитів, отримання відповідей та затвердження.
