## Огляд

`useProjectWorkspaces` надає read-only доступ до доступних робочих просторів проєктів. Файл існує, щоб повторно використовувати кешований перелік у межах прогону та безпечно обробляти збої: перехоплює помилки й не кидає винятків назовні.

## Поведінка

1. `useProjectWorkspaces` надає спільний стан доступних робочих просторів проєктів і ознаку активного завантаження.

2. Під час першого звернення завантажує перелік робочих просторів із налаштованих шляхів проєктів, щоб інтерфейс міг показати користувачу доступні варіанти для вибору.

3. Перетворює знайдені робочі простори у формат елементів списку з назвою для показу та шляхом як значенням.

4. Кешує результат у межах поточного прогону, щоб повторні звернення не виконували зайве сканування.

5. Скидає кеш після зміни шляхів проєктів, щоб наступне завантаження відображало актуальні робочі простори.

6. Дозволяє примусово оновити перелік робочих просторів, ігноруючи попередньо кешований результат.

7. У разі помилки залишає застосунок працездатним: фіксує збій у журналі, завершує стан завантаження та не передає виняток назовні.

8. Не змінює файлову систему або базу даних; лише читає доступні робочі простори для відображення в застосунку.

## Публічний API

- useProjectWorkspaces — зберігає спільний перелік mt-workspace, знайдених у шляхах проєкту користувача.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- Кешує результати в межах одного прогону.
