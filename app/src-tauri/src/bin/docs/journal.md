## Огляд

Цей бінарний інструмент є окремим, не пов'язаним з графічним інтерфейсом (GUI) модулем для роботи з журналом записів на рівні файлової системи для вузла MCP. Він реалізує логіку з `src/journal.rs`. Інструмент дозволяє створювати нові записи, завантажувати, оновлювати та перелічувати існуючі записи, використовуючи JSON для передачі намірів та даних змін. Операції виконуються через командний рядок (наприклад: `journal create '<json{intent,actor}>' | load <id> | update <id> '<patch>' | list`). Інструмент лише читає файлову систему та не здійснює запис. У випадку помилок він перехоплює їх (fail-safe), не вилучає винятки та повертає порожнє значення (наприклад, `null`) замість того, щоб кидати виняток. У разі помилки завершення роботи з кодом 2.

## Поведінка

1. Запускає бінарник, отримуючи список вхідних аргументів.
2. Визначає операцію за першим аргументом (`create`, `load`, `update`, `list`).
3. Якщо операція `create`, парсить JSON-дані для визначення наміру та актора. Запускає створення записи та виводить його згенерований ID у JSON.
4. Якщо операція `load`, витягує ID і викликає функцію для отримання відповідного запису.
5. Якщо операція `update`, витягує ID та JSON-пакет змін. Запускає оновлення запису за вказаним ID з використанням наданих змін.
6. Якщо операція `list`, викликає функцію для отримання списку всіх записів.
7. Якщо операція невідома, виводить повідомлення про помилку.
8. При успішному виконанні будь-якої операції виводить відповідний результат у форматі JSON у стандартний вивід.
9. При виявленні будь-якої помилки на етапі виконання або парсингу, виводить повідомлення про помилку у стандартний похибний вивід та завершує процес з кодом виходу 2.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
