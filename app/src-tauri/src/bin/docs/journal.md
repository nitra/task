## Огляд

Цей незалежний бінарний файл надає `request-journal` FS для node MCP bin, повторно використовуючи `src/journal.rs` без зв'язування зі стеком Tauri GUI (FS-in-Rust). Він реалізує командний інтерфейс для створення, завантаження, оновлення та переліку записів. При успіху він виводить JSON у stdout, а при помилці завершується з кодом 2. Система працює в fail-safe режимі, перехоплюючи помилки та повертаючи порожнє значення (наприклад, `null`) замість викидання винятків. Механізм кешування відсутній.

## Поведінка

1. Для команди `create` приймається JSON з наміром та актором. Система генерує ідентифікатор журналу, зберігає запис і виводить його разом з згенерованим ID у вигляді JSON.
2. Для команди `load` приймається ідентифікатор, і система шукає та виводить відповідний запис журналу.
3. Для команди `update` приймається ідентифікатор та JSON-патч. Система модифікує існуючий запис за цим ID та підтверджує успішне оновлення.
4. Для команди `list` система повертає повний список усіх записів журналу у вигляді масиву JSON.
5. Якщо вказана невідома операція, система виводить повідомлення про помилку.
6. Якщо будь-яка операція завершується помилкою, система виводить деталі помилки на stderr та завершує роботу з кодом виходу 2.
7. Якщо операція успішна, результат (запис, список, підтвердження) виводиться у вигляді JSON на stdout.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
