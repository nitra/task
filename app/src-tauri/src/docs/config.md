## Огляд

Конфігурація забезпечує єдине джерело правди (single source of truth) для шляхів до проєктів, які використовуються графічним інтерфейсом, внутрішнім агентом та бінарними файлами Node MCP. Шляхи зберігаються у `appLocalDataDir/config.json` у форматі `{ "project_paths": [...] }`. При відсутності чи порожній конфігурації автоматично повертається шлях за замовчуванням `~/www`. Механізм роботи є захищеним від помилок (fail-safe), не викидаючи винятків назовні, а повертаючи порожнє значення замість збою при певних помилках. Доступ до шляхів здійснюється через функції `get_project_paths` та `set_project_paths`.

## Поведінка

Поведінка:
get_project_paths повертає список шляхів до проєктів, зафіксованих у `config.json`, або повертає за замовчуванням шлях `~/www` у випадку відсутності або порожнього значення конфігурації.
set_project_paths зберігає наданий список шляхів до проєктів у `config.json`, створюючи необхідні каталоги, якщо вони відсутні.

## Публічний API

get_project_paths — Повертає шляхи до проектів, які шукає користувач; якщо не встановлені, за замовчуванням шукає у `~/www`.
set_project_paths — Зберігає шляхи до проектів, вказані користувачем.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
