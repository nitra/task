## Огляд

Цей модуль реалізує механізм для фіксації історії запитів (request journal), що зберігається як JSON-об'єкти у файлах `<requests_dir>/<id>.json`. Взаємодія з файловою системою здійснюється через Rust. Дані не видаляються; замість цього здійснюється лише патч-оновлення статусу або полів запису. Модуль надає інтерфейс для створення, завантаження, оновлення та перегляду цих записів, при цьому забезпечуючи відмовостійкість (fail-safe) шляхом перехоплення помилок та повернення `null` замість винятків.

## Поведінка

requests_dir визначає шлях до директорії, де зберігаються журнали запитів, використовуючи змінну оточення `TASK_REQUESTS_DIR` або шлях у папці підтримки застосунку.
create створює новий запис у журналі з початковим статусом "pending" та повертає його унікальний ідентифікатор.
load зчитує та повертає повний вміст запису за його ідентифікатором.
update застосовує часткове оновлення (`patch`) до існуючого запису та оновлює час його останньої модифікації.
list зчитує всі записи з директорії журналу та повертає їх список, відсортований за часом створення (від новіших до старіших).

## Публічний API

requests_dir — Знаходить каталог для зберігання запитів: або використовує змінну середовища `TASK_REQUESTS_DIR`, або локальний каталог даних програми.
create — Створює нову задачу зі статусом "очікує" та повертає її ідентифікатор.
load — Знаходить та завантажує задачу за її ідентифікатором.
update — Оновлює задачу, застосовуючи дані з наданого об'єкта та відзначаючи час останнього оновлення.
list — Виводить список усіх задач, розташованих у порядку від новіших до старіших.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
