## Огляд

Огляд. Файл надає структури для зберігання та отримання інформації про запусків CI з різних провайдерів.

Поведінка

PipelineRunSummary: структура для зберігання уніфікованого рядка статусу CI з провайдерів.
RunJob: структура для зберігання деталей окремого завдання у рамках прогону.
RunDetails: структура для деталізації виконання пайплайну, включаючи список джобів або загальний висновок.
list_pipeline_runs: отримує список усіх запусків CI з репозиторію, перевіряючи як GitHub, так і Azure Pipelines.
pipeline_run_details: отримує деталі одного запуску CI для визначеного провайдера, повертаючи відповідну структуру.
