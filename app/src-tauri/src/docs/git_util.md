## Огляд

Огляд
Файл використовує команду `git` для визначення кореня репозиторію. Виконує `git rev-parse --show-toplevel` з вказаною директорією. Перевіряє статус виконання команди `git` для визначення успішності. У разі помилки повертає помилку виконання. Успішно повертає об'єкт `PathBuf` зі шляхом. Файл не використовує кешування даних і не гарантує наявність репозиторію у вказаній директорії

## Поведінка

Поведінка

1. Використовує команду git для визначення кореня репозиторію
2. Виконує команду git rev-parse --show-toplevel з вказаним директорією
3. Перевіряє статус виконання команди git
4. Якщо статус не вказує на успіх, повертає помилку
5. У випадку помилки повертає помилку виконання
6. У випадку успіху повертає об'єкт PathBuf з отриманим шляхом
7. Не здійснює кешування даних
8. Не гарантує наявність репозиторію у вказаній директорії

## Публічний API

**repo_root** — визначає корінь репозиторію, виконуючи `git -C <tasks_dir> rev-parse --show-toplevel` для отримання кореневої директорії.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
