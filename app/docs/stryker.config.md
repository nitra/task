## Огляд

Цей файл конфігурує інструменти для мутаційного тестування (Stryker) та виконання тестів (Vitest). Він визначає, як будувати тестовий набір та аналізувати покриття. Конфігурація спирається на `mutation.json` та `incremental.json` для визначення параметрів мутацій та увімкнення інкрементального режиму виконання тестів.

## Поведінка

1. Конфігурація Stryker для тестування визначається на основі `stryker.config.mjs`.
2. Для виконання тестів використовується `vitest`, з посиланням на `vitest.config.js`.
3. Аналіз покриття виконується для кожного окремого тесту, що забезпечує швидший збір результатів порівняно з повним запуском тестового набору на кожен мутант.
4. Репортери для збереження результатів встановлюються у форматах JSON та clear-text; JSON-звіт зберігається у `reports/stryker/mutation.json`.
5. Для забезпечення стійкості та прискорення повторних запусків, результати зберігаються між виконаннями у файлі `reports/stryker/incremental.json`.
6. Процес мутації ігнорує Vue `<script setup>`-макроси (такі як `defineProps`, `Emits`, `Model`, `Slots`, `Expose`, `Options`) за допомогою локального плагіна для уникнення помилок компіляції SFC.
